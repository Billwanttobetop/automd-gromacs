## 五、支持的加速模式

### 5.1 Dihedral aMD

**适用场景**:
- 蛋白质构象变化
- 肽链折叠
- 侧链旋转

**优点**:
- 针对性强
- 稳定性好
- 易于重加权

**缺点**:
- 仅加速二面角
- 对整体运动效果有限

**参数推荐**:
```bash
AMD_TYPE=dihedral
AMD_E_DIHED=auto      # 自动计算
AMD_ALPHA_DIHED=auto
```

### 5.2 Total Energy aMD

**适用场景**:
- 小分子
- 配体结合
- 全局构象搜索

**优点**:
- 全局加速
- 探索范围广

**缺点**:
- 可能过于激进
- 稳定性较差
- 重加权复杂

**参数推荐**:
```bash
AMD_TYPE=total
AMD_E_TOT=auto
AMD_ALPHA_TOT=auto
```

### 5.3 Dual-Boost aMD

**适用场景**:
- 复杂体系
- 蛋白质-配体复合物
- 需要最强加速

**优点**:
- 平衡性好
- 加速效果最强
- 适用范围广

**缺点**:
- 参数调优复杂
- 计算开销稍大
- 重加权最困难

**参数推荐**:
```bash
AMD_TYPE=dual
AMD_E_DIHED=auto
AMD_ALPHA_DIHED=auto
AMD_E_TOT=auto
AMD_ALPHA_TOT=auto
```

## 八、已知限制

### 8.1 技术限制

1. **PLUMED依赖**
   - GROMACS原生不支持aMD
   - 需要PLUMED或使用metadynamics替代

2. **重加权复杂性**
   - 大偏置 (σ_boost > 10 kT) 时重加权困难
   - 需要高级方法 (Cumulant展开、MBAR)

3. **参数敏感性**
   - 参数不当可能导致系统不稳定
   - 需要预运行和谨慎调优

4. **收敛性**
   - 需要足够长的模拟时间
   - 收敛性评估需要手动分析

### 8.2 适用范围

**适用**:
- ✅ 蛋白质构象变化
- ✅ 配体结合路径
- ✅ 全局构象搜索
- ✅ 不知道反应坐标的情况

**不适用**:
- ❌ 需要精确自由能 (用Umbrella/Metadynamics)
- ❌ 化学反应 (需要QM/MM)
- ❌ 系统过大 (> 100k原子)
- ❌ 需要原子级精度
